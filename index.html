py-chess-arena

create, battle, and analyze chess bots

About

py-chess-arena is a Python library for creating, battling, and analyzing chess bots. The API makes it easy to make your own bot. You can also test your bots against the included py-chess-arena combatants.


Getting Started

Requirements

  • Python 2.7
  • python-chess

To clone the git repo:

~/> git clone https://github.com/OldGregg570/chess-ai-battle.git

Implementing a Chess AI

Implementing a new AI combatant is simple. An AI must implement the get_move() method in order to compete. This method accepts the current board state. 

~/chess-ai-battle/> 
from src.ai.Combatants import Combatant
import random

class Randy(Combatant):
def get_move(self, board):
    # get the list of legal moves from the current board state
    legal_moves = list(board.legal_moves)

    # return the random legal move
    return legal_moves[random.randint(0, len(legal_moves) - 1)]

Once a new Combatant has been implemented, it can be used in a chess battle. The return value for a battle is a tuple with three values: (stalemates, white_wins, black_wins) 

~/chess-ai-battle/> 
import src.Battle

trial_count = 5
p1, p2 = Randy, Randy

print Battle(trial_count, p1, p2).run()

# example output
>>> (5, 0, 0)

print Battle(1000, p1, p2).run()

# example output
>>> (732, 133, 135)

As you can see here, Randy tends to stalemate a lot when he plays with himself. In fact, A Randy vs Randy battle can been expected to stalemate around 70% of the time as observed with py-chess-arena
